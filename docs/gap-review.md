# Submail 功能复盘与后续路线

## 本轮已经完成

### 邮件核心

- 多邮箱 IMAP/POP3/SMTP 配置、连接测试、凭据加密和账号级访问控制；POP3 使用 UIDL 去重且不执行服务器删除。
- INBOX UID 增量同步、定时调度、重试恢复、并发限制、运行日志和正文/附件上限。
- 文本/HTML、CC/BCC、附件、Reply-To 与线程头发送；已发送本地落库。
- API 发送幂等、请求摘要冲突检测、主体与账号命名空间隔离、失败退还每日额度。
- SMTP/IMAP 的 STARTTLS 强制、TLS 1.2+、连接/握手/socket 超时和安全错误映射。
- 草稿、回复/转发、星标、归档、垃圾邮件、垃圾箱、附件管理和 SQLite FTS5 搜索。

### AI 与翻译

- 管理后台配置任意 OpenAI-compatible 服务，API Key 加密保存并遮罩返回。
- 邮件总结、推荐回信和邮件生成；生成结果只写入编辑器，不自动发信。
- 消息型 AI/翻译同时要求邮件读取权限，避免仅凭 AI scope 读取邮件派生内容。
- 默认 Google、LibreTranslate 和自定义翻译服务；长文本完整分块，不静默截断。
- 固定的 prompt-injection 防护前缀，以及 401、429、超时等安全上游错误类型。

### MCP 与外部调用

- stdio 和远程 Streamable HTTP MCP，共 8 个邮件、AI、翻译工具。
- 每次 HTTP 请求独立 Bearer Key，不依赖容器内固定全局 Key。
- Key 支持 scope、账号白名单/显式全部账号、过期、撤销和每日发信额度。
- 搜索在 SQL/FTS 层执行账号约束；API Key 的列表结果不暴露正文。
- Origin/Host/请求体/超时限制和不含正文、地址、附件数据的 MCP 审计。

### 部署、安全与运维

- Docker Compose 一键部署；API/MCP 仅内网，Web 同源网关默认只绑定回环地址。
- 非 root、只读根文件系统、健康检查、restart、graceful shutdown 和请求排空。
- 强生产配置校验、首个管理员原子初始化、登录/发送/AI/翻译限流、同源反向代理与安全响应头。
- AES-GCM 密文版本/AAD、旧密文兼容、主密钥指纹错误提示和敏感字段日志清理。
- SQLite 在线备份、流式 SHA-256、完整性校验、运行锁、WAL 一致回滚副本和原子恢复。
- SQLite、Compose 内置 MySQL、外部 MySQL 共用异步仓储/事务层；MySQL 已通过自动建表、认证、草稿、搜索、设置 UPSERT、配额事务和幂等冲突测试。
- Redis + BullMQ 持久化队列；手动/定时同步与 API/MCP SMTP 发信统一分发，健康检查同时检查数据库和队列。
- 审计/同步记录保留清理、数据库目录权限和异常中断同步状态修复。
- TypeScript 全工作区构建、API 集成测试、Docker/MCP SDK/浏览器烟测及真实邮件、AI 手工接入验证。

## 仍需完善

### P1：数据库与邮件一致性

1. **跨数据库迁移与 MySQL 全文索引**：SQLite/MySQL 均可新部署，但尚无一键搬迁工具；MySQL 搜索当前使用兼容 LIKE，未启用供应商特定 FULLTEXT。
2. **IMAP 状态写回**：Web 的已读、星标、归档、删除目前是本地状态，尚未写回远端邮箱。
3. **多文件夹同步**：已支持服务商可发现的 Sent、Drafts、Junk/Spam、Trash 和 Archive 特殊目录读取；自定义文件夹、远端写回与冲突处理尚未完成。
4. **OAuth connector**：Gmail、Microsoft 等 OAuth2 尚未实现；常见服务商的应用专用密码/客户端授权码引导和推荐服务器参数已提供，但强制 OAuth 的企业账号仍无法接入。

### P1：生产投递与存储

1. **投递状态面板与死信处理**：Redis 已持久化任务，但尚无后台队列面板、人工重放和告警；SMTP 为防重复投递默认不自动重试。
2. **部分收件人投递跟踪**：API 会返回 accepted/rejected 并只把实际接受地址写入 Sent，但尚无专门的投递结果面板和标准 DSN 处理。
3. **DKIM/策略能力**：依赖上游邮箱服务器处理 SPF/DKIM/DMARC；Submail 本身还不签名。
4. **附件外部存储**：附件目前存 SQLite BLOB，大规模部署需要本地文件/S3 适配、生命周期和容量告警。
5. **恶意附件防护**：尚无病毒扫描、Office/PDF 安全沙箱和内容净化。

### P2：管理与体验

1. 主密钥轮换流程、MCP / API Key 一键轮换和 IP/CIDR 白名单。
2. 多语言 i18n、HTML 富文本编辑器、模板/签名/发件别名。
3. 手机 App 所需的推送、离线缓存和专用分页 API。
4. 更完整的监控指标、告警、队列可视化和管理员审计导出。

### 测试缺口

- 将 API、MCP 协议、备份恢复和浏览器端到端测试纳入 CI。
- 增加大邮箱/大附件压力测试、断网恢复、慢 SMTP、进程关机中发送和数据库故障注入。
- 对不同 IMAP 服务商建立不含真实凭据的可重复兼容性测试矩阵。

## 推荐实施顺序

1. SQLite/MySQL 跨库迁移、MySQL FULLTEXT 与备份自动化。
2. IMAP 状态写回、自定义文件夹同步和冲突策略。
3. 队列投递跟踪、死信处理和告警。
4. 附件文件/S3 存储、保留策略与病毒扫描。
5. OAuth connector、富文本模板和移动端能力。
